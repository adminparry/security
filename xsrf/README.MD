### xsrf
跨站脚本攻击

### 核心
攻击者：获得用户状态接口服务的操作权限去执行一定的动作
防范者：为自己服务接口为自己当前用户所用

### 场景

被害者：
www.bhz.com
页面
登录状态写入cookie
服务接口 
add 新增 
edit 修改 
del 删除

谋害者：
www.mhz.com 
页面
iframe src = www.bhz.com

获得了登录状态去调用www.bhz.com的服务接口

如果想监控这个登录状态只能选择轮询

### 防范

给用户一个唯一的token进行验证

攻击者可以获取登录的状态但获取不到登录者是用哪个token去登录的
